
🚀 Microservices Deployment Guide (HNG Stage 2)This project is a fully containerized 3-tier application featuring a FastAPI backend, a Node.js frontend, a Python-based Worker, and Redis for state management and job queuing.📋 PrerequisitesBefore starting, ensure the following are installed:Git (to clone the repository)Docker (Latest stable version)Docker Compose v2.0+Trivy (Optional, for local security auditing)🛠️ Setup Instructions1. Clone the repositoryBashgit clone https://github.com/Lucky059/hng14-stage2-devops.git
cd hng14-stage2-devops
2. Configure Environment VariablesThe stack now uses a centralized networking and password strategy. Create .env files in each service directory using the placeholders below:Root .env & api/.env & worker/.env:Code snippetREDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=hngpassword
APP_ENV=development
QUEUE_NAME=job
frontend/.env:Code snippetAPI_URL=http://api:8000
Security Note: .env files are excluded from Git. Refer to .env.example for the full schema.📦 Deployment & Integration3. Build and Start the StackThis command builds the images locally and orchestrates the dependencies (ensuring Redis is healthy before the API starts).Bashdocker compose up --build -d
4. Verify System HealthOur CI/CD pipeline asserts health via the following endpoints:ServiceEndpointExpected OutputAPIhttp://localhost:8000/health{"status":"ok"}Frontendhttp://localhost:3000HTTP 200 (Dashboard Load)Redisdocker exec -it redis redis-cli -a hngpassword pingPONG🛡️ CI/CD Pipeline & SecurityThe repository is governed by a 6-stage GitHub Actions pipeline (.github/workflows/ci.yml):Lint: Validates Dockerfiles (Hadolint) and Python syntax (Flake8).Test: Executes unit tests with pytest and generates coverage reports.Build: Packages microservices into Docker images.Security Scan: Uses Trivy to audit the API image for CRITICAL vulnerabilities.Integration Test: Spins up the full stack in the GitHub Runner to verify inter-service communication.Deploy: Simulates a rolling update for the production environment.Local Security AuditingTo run the same scan locally as the CI:Bashtrivy image hng-api:latest --severity CRITICAL --ignore-unfixed
✅ Successful Startup Checklist[ ] docker compose ps shows all services as Up (healthy).[ ] Submitting a task via the Frontend dashboard updates the Redis state.[ ] Worker logs show job consumption: docker compose logs -f worker.[ ] No Connection Refused errors in the API logs regarding Redis.
